# 📋 과제전형 연습 — 포인트 시스템 API

> 2티어 B2B SaaS 백엔드 과제전형 수준 / 권장 소요 3~5일

---

## 배경

회원이 적립금(포인트)을 적립하고 사용하는 시스템의 백엔드를 구현하세요.
실제 커머스·핀테크에서 쓰이는 핵심 도메인입니다.

---

## 필수 요구사항

### 1. 포인트 적립
- 회원에게 포인트를 적립한다
- 적립 건마다 **만료일이 있다** (적립일로부터 365일 후 자정 만료)
- 1회 적립 한도와 개인 최대 보유 한도가 있다
  - 한도 값은 직접 정하고 README에 명시

### 2. 포인트 사용
- 회원이 보유 포인트를 사용한다
- **만료가 임박한 적립 건부터 차감**한다
- 잔액이 부족하면 사용에 실패한다
- 한 번의 사용이 여러 적립 건에 걸쳐 차감될 수 있다

### 3. 포인트 사용 취소
- 특정 사용 건을 취소하면 차감했던 포인트를 복원한다
- ⚠️ 복원 대상 적립 건이 이미 만료됐다면?
  → **정책을 직접 결정하고 README에 근거를 작성**

### 4. 포인트 적립 취소 (관리자)
- 잘못 적립한 건을 취소한다
- ⚠️ 이미 사용된 적립 건이라면?
  → **정책을 직접 결정하고 README에 작성**

### 5. 잔액 조회
- 회원의 현재 사용 가능한 총 잔액을 조회한다

## 💡 출제자 힌트 (실제 과제엔 없는 학습용)

이 과제엔 **함정 3개**가 의도적으로 심어져 있습니다:

1. **만료 차감 순서** — "만료 임박 순 차감"을 어떻게 구현하는가 = 도메인 모델링 실력
2. **취소 시 만료 처리** — 정답이 없음. 결정하고 **근거를 쓰는 것**이 시니어 증명
3. **동시성** — 락 전략을 README에 설명하면 가산점
