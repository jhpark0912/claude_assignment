# db 구성

## 회원 테이블
- user_id: 유저 id(pk) - string
- username: 유저 이름 - string
- password: 비밀번호 (마스킹) - string
- minus_point: 취소 적립 포인트 - int
- date_created: 생성일(yyyy-MM-dd)
- date_updated: 수정일(yyyy-MM-dd)

## 회원 포인트 테이블
- id: 포인트 id 값(pk) - int
- user_id: 유저 id (fk) - string
- point: 포인트 (int)
- expired_date: 만료일자(yyyy-MM-dd)
- balance: 잔여 포인트(int)
- date_created: 생성일(yyyy-MM-dd)
- date_updated: 수정일(yyyy-MM-dd)

## 포인트 사용 내역
- id: 사용 내역 id 값(pk) - int
- user_id: 유저 id (fk) - string
- point : 사용한 포인트 (int)
- date_used : 사용된 날자 (yyyy-MM-dd)
- date_cancled: 취소된 날짜

## 사용 내역과 포인트 매핑 테이블
- id: 매핑 id
- point_id: 회원의 적립한 포인트 id(fk)
- point: 사용된 point


# 규칙

- 해당 규칙은 README에 작성함

## 최대 포인트
- 1일 최대 포인트 10000(1만) 포인트
- 유저 최대 포인트 1000000(100만) 포인트

## 포인트 만료일자
- 포인트 등록일로부터 365일 자정 만료 (expired_date 의 자정 예, 2026-12-31 인 경우 2027-01-01 00:00 만료)

## 포인트 사용규칙
- 포인트는 만료일로부터 가까운 정보 기준으로 사용
- 사용하고자 하는 잔액이 모자라면 사용에 실패한다. 잔액에 대한 계산을 제일먼저 진행하고 실패처리한다.
- 사용시 여러개의 적립된 포인트를 각 차감한다. 단, 앞서 규칙 대로 만료일 가까운 정보 기준 사용
- 포인트 사용과 함께 포인트 사용내역에 하나의 열이 추가된다.
- 유저의 전체 잔여 포인트를 만료일자가 적게 남은 순으로 데이터를 가져오며, 잔여 포인트 만큼 사용을 하여 계산한다. 이를 사용 내역과 포인트 매핑 테이블에 저장한다.
- 잔여 포인트는 사용한 포인트만큼 제거하고, 이보다 아직 사용할 포인트가 남으면 다음 포인트에서 계속 차감한다.

## 포인트 취소규칙
- 포인트를 사용한 취소는 취소시에 사용내역 기준으로 취소를 진행하며, 매핑테이블의 모든 사용된 항목을 전환한다.
- 취소시, 만약 만료된 포인트가 있는 경우 해당 포인트의 환급은 없다. 
    - 그 이유는 이미 해당 포인트는 환불이 되고 사용이 되어야 하는 요소이지만, 만료가 되었다는 것은 이미 사용이 불가능한 포인트이기 때문
    - 사용 내역과 포인트 매핑 테이블 기준으로 복원하며, 포인트 사용내역 테이블에는 취소 일자를 기록한다.

## 잘못 적립된 포인트의 경우
- 잘못 적립된 건에 대해서 관리자가 취소하는 경우 해당 취소 적립 포인트를 유저에 매핑하여 저장한다.
- 이후 포인트 적립 시에 해당 취소 적립 포인트에서 차감하도록 처리한다.
