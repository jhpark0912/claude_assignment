# 🎯 Training Point — 과제전형 연습 레포지토리

> Claude를 활용해 **실전형 과제를 출제받고, 직접 풀어나가는** 학습 레포지토리입니다.

---

## 📌 이 레포는 무엇인가요?

실제 채용 과제전형(코딩 테스트·백엔드 과제)을 혼자서도 반복 연습하기 위한 공간입니다.

일반적인 과제 연습과 달리, 이 레포는 **Claude(AI)를 출제자이자 피드백 제공자로 활용**합니다.

```
┌─────────────┐   ① 과제 출제    ┌─────────────┐
│             │ ───────────────▶ │             │
│   Claude    │                  │     나      │
│  (출제·평가) │ ◀─────────────── │   (풀이)    │
└─────────────┘   ② 제출·피드백   └─────────────┘
```

1. **출제** — Claude가 실무 수준의 과제를 제시합니다. (요구사항·제약조건·평가 기준 포함)
2. **풀이** — 내가 직접 설계하고 구현하며, 막히는 지점은 학습 차원에서 Claude에게 질문할 수 있습니다.
3. **제출·피드백** — 완성한 코드를 Claude에게 제출하면 종합 판정, 루브릭별 점수, 면접 예상 질문까지 피드백을 받습니다.

> 💡 실제 과제전형에서는 중간에 질문할 수 없지만, **여기서는 학습이 목적**이므로 자유롭게 묻고 배우며 진행합니다.

---

## 🗂️ 레포 구조

과제는 `NN.과제명/` 형태의 폴더 단위로 관리됩니다. 번호 순서대로 난이도·주제가 쌓여 갑니다.

```
04.training-point/
├── README.md                  ← 현재 문서
├── 01.point_system/           ← 과제 #1: 포인트 시스템 API
│   ├── 과제_포인트시스템_API.md   ← 과제 명세 (요구사항·평가 기준)
│   └── 구조설계.md               ← 풀이를 위한 설계 메모
└── ...                        ← 이후 과제가 순차적으로 추가
```

각 과제 폴더는 다음을 포함합니다.

| 파일 | 설명 |
|------|------|
| `과제_*.md` | Claude가 출제한 과제 명세 (배경·요구사항·제약·평가 기준) |
| `구조설계.md` | 내가 작성하는 설계 메모 (DB 구성·정책 결정 등) |
| 구현 코드 | 과제 요구사항에 따른 실제 소스 및 테스트 코드 |

- **상태**: 🚧 진행 중

## 🚀 진행 방법

1. **과제 확인** — 해당 과제 폴더의 `과제_*.md`를 읽고 요구사항·평가 기준을 파악합니다.
2. **설계** — `구조설계.md`에 도메인 모델·정책 결정·트레이드오프를 정리합니다.
3. **구현** — 요구사항에 맞게 코드를 작성합니다. (오버엔지니어링은 지양)
4. **테스트** — 단위/통합 테스트로 엣지 케이스를 검증합니다.
5. **커밋** — 의미 단위로 커밋해 히스토리를 남깁니다.
6. **제출·피드백** — Claude에게 제출하고 피드백을 받아 개선합니다.

## 🧭 학습 원칙

- ✅ **요구사항에 딱 맞게, 깔끔하게** — 과한 추상화·MSA·이벤트소싱은 지양

- ✅ **의미 단위 커밋** — 작업 흐름이 드러나는 히스토리
